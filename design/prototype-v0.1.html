v0.1 切片边界

✅ 要做的

  • Tauri 壳 + macOS only · < 100 MB / < 1 s 冷启动
  • 左上 Agent 维度: 飞书 IM / 邮箱 (消息渠道) · 日程 / TODO (工作流) · 知识库 (Skills/Memory/错误码.md) · 🏪 技能商店
  • 左下会话/工作区统一列表 · 支持从 chat promote 成 workspace
  • 中间: 单 tab 模式——[+] 在左, Silent Chat 固定最右, 用户从 [+] 开浏览器/终端/文件 tab,往左插入挤压
  • Silent Chat pane 内: Push (上) + 问答 (下)
  • 内嵌浏览器 (WKWebView + JS 注入) · P0 唯一观察通道
  • "教教我" 3 问 → skills/*.yaml
  • Skill v1 教学模式: 一步一停,每步带 [继续/改/停];连续顺利 3 次 → 升级 auto
  • 错误码.md 作为知识库文件(一个.md),bytedcli 本地封装 + Claude 语义兜底

⏭ 推到 v0.2 的

  • 文件 watcher + 终端 hook (后续两条观察通道)
  • 双向飞书 (agent 主动推回飞书 · 外部回复事件 hook)
  • Skill 信任等级自动升降 (v1 教学 → v2 auto → v3 可后台 cron)
  • Pattern 检测算法 (MVP 用 LLM 摘要)
  • Session → Workspace promote 的落盘切换
  • 技能商店 (skill 导入/发布/订阅)
  • 日程 / TODO 独立视图 (MVP 只是入口,V2 做内容页)

❓ 待澄清

  • 第一次手动执行时, AI 辅助 vs 用户自己操作的比例 (agent 多激进?)
  • 知识库文件 (错误码.md 等) 初始化: 人工导入 / bytedcli 拉取 / 云端同步?
  • skill 教学模式每步确认的粒度是否太细 (会不会烦)?
  • "连续 3 次顺利升级 auto" — 顺利的定义 (用户没改 = 顺利?)
  • skill 多版本管理 (v1 → v2 何时触发重新教)
  • 技能商店里别人的 skill 导入后,要不要也走一次教学模式?
