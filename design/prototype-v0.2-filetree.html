关键设计点
1. LeftNav 职责 = 脑子切换(agent 维度 / session 切换) —— 保持现有 sections
2. 文件树职责 = 工作心流 —— session 特定,点 session 的 ws tip 才进入
3. 触发交互:点 session 行右侧 ws tip 或 tab bar 最左 📁 icon → LeftNav 窄化(~96px) + 文件树 panel(220px) 插入
4. 关闭:再点 tab bar 的 📁 icon → 文件树收起、LeftNav 恢复宽。Icon 常驻 tab bar 作为唯一 toggle
5. 切换 session:文件树自动 follow active session 的 workspace path
6. 窄 LeftNav:1 icon + 2 字密度,顶部继续显示消息渠道/工作流/知识库入口;中间一条 1px 分割;底部会话(无 icon,仅文字)。顶部 « 展开 回到完整模式
7. 浏览器几乎不挤压:256 → 96+220=316,中间工作区减 60px(相对 1400 窗宽 <4%,可接受)
